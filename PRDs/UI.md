# 物品价值记录网页版 UI 设计文档
**文档版本**：V1.0  
**产品形态**：Web 端  
**设计范围**：当前正式网页版 UI  
**设计目标**：围绕“物品记录、持有时间、日均成本、提醒管理、统计分析”构建一套完整、统一、易用的界面系统。

---

# 1. UI 设计目标
本产品的 UI 不是传统后台系统，也不是纯内容型产品，而是一套偏生活管理的工具界面。  
界面设计的核心目标有四个：

## 1.1 让用户快速开始记录
用户进入系统后，应立刻知道：

+ 这是做什么的
+ 从哪里新增物品
+ 去哪里看物品列表
+ 提醒和统计在哪里查看

## 1.2 让核心数据一眼可懂
本产品最重要的信息包括：

+ 购买价格
+ 购买日期
+ 持有天数
+ 日均成本
+ 当前状态
+ 提醒状态

这些信息必须具备高可读性和清晰层级。

## 1.3 让管理路径顺手
用户在网页端最常做的动作包括：

+ 新增物品
+ 搜索物品
+ 筛选分类
+ 查看提醒
+ 打开物品详情
+ 导出数据
+ 切换深色模式

这些操作都应放在直观位置，不应隐藏过深。

## 1.4 保持整体统一
所有页面在视觉风格、组件规范、间距、按钮、标签、弹窗、表格样式上都要保持统一，避免页面间割裂。

---

# 2. 设计原则
## 2.1 清晰优先
信息层级必须比装饰性更重要。

## 2.2 轻量优先
使用简洁布局、适度留白、低干扰视觉，降低使用压力。

## 2.3 数据优先
价格、日期、成本、提醒等数据必须清楚突出。

## 2.4 一致优先
组件风格、按钮层级、弹窗样式、状态标签在全站统一。

## 2.5 易扩展优先
本期虽然不做复杂功能，但页面结构和组件体系必须支持后续扩展。

# 7. 深色模式 UI 设计
## 7.1 功能定位
满足长时间使用和不同环境下的视觉偏好。

## 7.2 切换入口
建议放在：

+ 顶部区域右侧
+ 设置页中

## 7.3 深色模式原则
深色模式不只是反色，应保持层级和可读性。

### 深色模式建议
+ 页面背景：深灰黑
+ 卡片背景：中深灰
+ 文字：浅灰白
+ 分隔线：低对比灰
+ 主按钮：白字深色按钮或高对比主色按钮
+ 状态标签：保持色相，但降低刺眼程度

## 7.4 深色模式要求
+ 数据数字仍要清晰突出
+ 状态标签保持可辨识
+ 输入框边界不能消失
+ 表格表头和行分隔依旧清楚

# 8. 右下角提醒小弹窗设计

## 8.2 展示位置

+ 编辑成功

+ 导出成功
+ 导出失败

+ 标记完成提示

## 8.4 设计形式
采用 Toast / 小弹窗样式：

+ 小尺寸
+ 带标题
+ 可带简短说明
+ 自动消失
+ 支持手动关闭

## 8.5 状态类型
+ 成功
+ 提示
+ 警告
+ 错误

## 8.6 设计要求
+ 不遮挡主要操作区
+ 不同时出现过多条
+ 支持轻微堆叠
+ 自动消失时间建议 2~4 秒

# 15. UI 设计总结
当前这版 UI 的核心方向是：

**做一套完整、统一、轻量、适合长期使用的生活物品管理网页界面。**

它重点解决的是：

+ 让用户快速记录
+ 让数据容易理解
+ 让提醒更自然出现
+ 让管理动作更顺手
+ 让界面在浅色和深色模式下都稳定可用

这版不再包含：

+ 图片上传交互
+ AI 录入引导
+ 浏览器通知

这版明确包含：

+ 独立物品详情页
+ 数据导出
+ 深色模式
