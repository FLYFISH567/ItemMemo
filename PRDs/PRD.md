## 1. 产品概述
### 1.1 产品名称
物品价值记录（网页版）

### 1.2 产品简介
本项目网页版是一套围绕“生活物品记录、持有时间计算、日均成本分析、提醒管理”打造的轻量化管理平台。  
用户可以通过网页端快速录入和管理日常生活中的各类物品，并查看每件物品的购买信息、持有时间、日均成本以及提醒状态，从而更清楚地了解物品的使用价值和更换需求。

### 1.3 产品目标
以更低的开发成本和更快的迭代速度，先完成产品概念验证，帮助用户建立“记录物品—计算成本—接收提醒—查看统计”的完整闭环。

### 1.4 产品定位
这是一个帮助用户管理生活物品、量化物品使用价值、并提供更换与补购提醒的网页版工具。

## 2. 项目背景与价值
### 2.1 背景
现有工具大多分为两类：

+  记账工具：关注花了多少钱，但不追踪物品后续价值 
+  收纳工具：关注分类与整理，但缺少成本分析与提醒能力 

用户在购买物品后，通常只记得“买了什么、花了多少钱”，却很少持续关注：

+  已持有多久 
+  平均每天成本多少 
+  是否长期闲置 
+  是否到期需要更换 
+  是否即将消耗完需要补购 

### 2.2 核心价值
本产品的核心价值不在“记消费”，而在“看价值”。

通过把购买价格和购买时间转化为可持续观察的指标，帮助用户：

+  更理性地理解消费 
+  更系统地管理生活物品 
+  更及时地处理更换和补购 
+  更直观地发现高价值物品和低效购买

## 3. 目标用户
### 3.1 用户类型
1.  理性消费用户  
希望知道买的东西到底值不值，愿意做消费复盘。 
2.  精致生活 / 收纳管理用户  
有整理习惯，愿意把生活物品数字化管理。 
3.  数码 / 家居 / 装备爱好者  
购买金额高、物品使用周期长，更关心长期价值。 
4.  年轻上班族 / 学生  
需要一个轻量工具帮助自己管理生活和消费。 

### 3.2 用户痛点
+  买了很多东西，后面不清楚是否常用 
+  只知道总价，不知道长期摊下来每天花了多少 
+  容易忘记更换周期和补购时间 
+  物品多时查找困难，记录零散 
+  缺少一个兼顾“物品管理 + 成本分析 + 提醒”的工具

### 4.2 成功标准

+  用户能理解并查看持有时间与日均成本 
+  用户能通过提醒中心关注更换/补购事项 
+  用户能在列表、分类、统计中快速找到信息 
+  产品闭环完整：录入 → 查看 → 提醒 → 复盘

## 8. 功能需求说明
## 8.1 首页总览功能
### 功能说明
首页展示用户当前最重要的信息，包括总体数据、近期提醒和最近记录。

### 输入
系统从本地数据读取物品、分类、提醒信息。

### 输出
展示统计卡片、最近记录列表、提醒摘要、分类概览。

### 交互要求
+  点击“新增物品”可弹出新增弹窗 
+  点击侧边栏可切换页面 
+  首页中的列表和提醒卡片未来可支持跳转详情 

---

## 8.2 分类管理功能
### 功能说明
支持系统默认分类和用户自定义分类管理。

### 输入
用户输入分类名称。

### 输出
生成新分类并加入分类列表。

### 规则
+  分类名称不能为空 
+  分类名称建议唯一 
+  删除分类时需进行二次确认 

---

## 8.3 物品管理功能
### 功能说明
支持新增、查看、编辑、删除生活物品记录。

### 输入字段
+  name 
+  category 
+  price 
+  purchaseDate 
+  reminder 
+  note 
+  status 

### 状态类型
+  使用中 
+  闲置中 
+  即将更换 
+  补购提醒 
+  已更换（后续） 
+  已消耗完（后续） 
+  已丢弃（后续） 

### 输出内容
生成一条物品记录，并在首页、列表、统计中同步可见。

---

## 8.4 持有时间计算功能
### 规则
持有天数 = 当前日期 - 购买日期 + 1

### 补充规则
+  当天购买，按 1 天计算 
+  日期无效时，兜底按 1 天计算 
+  未来日期录入时，最小仍按 1 天处理 
+  后续若支持“结束日期”，可用结束日期替代当前日期参与计算 

---

## 8.5 日均成本计算功能
### 规则
日均成本 = 购买价格 ÷ 持有天数

### 补充规则
+  价格为空或 0 时，结果为 0 
+  结果保留两位小数 
+  按持有天数实时计算 
+  新增物品后自动计算，不依赖手工输入 

---

## 8.7 提醒管理功能
### 提醒类型定义
1.  固定周期提醒  
用于有明确更换周期的物品，如牙刷、刀头、护肤品等。 
2.  持有时间提醒  
当持有时间达到阈值时提示关注。 
3.  高日均成本提醒  
价格较高但持有时间较短，提示关注物品使用效率。 
4.  补购提醒  
针对消耗型物品，提醒及时补购。 

### 当前阶段实现
+  首页提醒卡片 
+  提醒中心列表 
+  提醒状态展示 
+  标记完成 / 稍后提醒按钮（原型级） 

### 后续扩展
+  浏览器通知 
+  邮件提醒 
+  自动计算提醒日期

## 10. 本地存储方案
### 10.1 存储方式
网页版当前基于本地保存，建议使用：

+ `localStorage` 作为 MVP 数据存储方案 

### 10.2 存储对象
+  categories 
+  items 
+  reminders 
+  settings 

### 10.3 优势
+  开发成本低 
+  不依赖后端 
+  适合概念验证 

### 10.4 局限
+  无法跨设备同步 
+  清除浏览器缓存后可能丢失数据 
+  不适合多用户使用

## 11. 交互流程
## 11.1 新增物品流程
进入页面 → 点击“新增物品” → 填写表单 → 校验必填字段 → 保存 → 自动计算持有时间和日均成本 → 写入本地数据 → 刷新首页与列表展示

## 11.2 管理分类流程
进入分类管理页 → 查看现有分类 → 新增/编辑/删除分类 → 本地保存 → 同步更新分类展示

## 11.3 查看提醒流程
进入首页或提醒中心 → 查看待处理提醒 → 点击“标记完成”或“稍后提醒” → 更新提醒状态

## 11.4 查看统计流程
进入统计页 → 查看总量、分类分布、洞察文案 → 作为后续优化的使用依据
